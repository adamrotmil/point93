Accessibility Requirements (applies to all panels)
• Color + text + shape: Never rely on color alone. All band indicators use triple redundancy: color + text label + shape. Shape system: ● Top Tier (filled circle), ◐ Competitive (half circle), ▲ Needs Work (triangle), ◇ At Risk (diamond). CSS classes: .band-shape-top, .band-shape-comp, .band-shape-needs, .band-shape-risk.
• 10% colorblind: ~10% of males are red-green colorblind. Green/red pair (Top Tier/At Risk) must always include text labels.
• Contrast ratios: All text meets WCAG AA (4.5:1 for body, 3:1 for large text). Muted text at var(--text-muted) #5c6478 passes on white backgrounds.
• Touch targets: All interactive elements minimum 44×44px (WCAG 2.5.5). Card CTAs, nav items, and buttons all meet this.

Module List View Rules — §5.1
• Landing page when advisor clicks "Assessments" in nav
• All 17 modules organized by their 6 practice areas
• Each card shows: module name, area code, data point count, one-sentence description, status, score + band if completed
• Completed cards include DP Dot Array + Hover Detail: Row of band-colored dots (one per DP) with a persistent detail box to the right. On hover, dot scales up and box shows DP name + band-colored score. Creates diagnostic curiosity without revealing the full picture.
• In Progress cards show progress bar (questions answered / total) + "Continue Assessment →" CTA
• Not Started cards show dash placeholder + "Start Assessment" implied by click
• Clicking a card navigates to Module Home (returning) or directly to questions (first-time per §10.3)
• Module Gallery layout per DD-1: grid of cards chosen for scanability and area grouping
Completed Card — DP Dot Array + Hover Detail
• Score + Band — Score circle color-coded by band. Band label next to score.
• Module Description — One-sentence diagnostic framing of what the module measures. Present on all cards (completed, in progress, not started).
• DP Dot Array — Row of 7px circles, one per data point. Each dot colored by band: ● Top Tier, ● Competitive, ● Needs Work, ● At Risk. Modules with 13+ DPs use flex-wrap.
• Hover Detail Box — Persistent container sits to the right of the dot row. On hover, the active dot scales up (1.6×), other dots dim (35% opacity), and the detail box shows: band-colored score (11px, bold) + DP name (9px). When no dot is hovered, shows "← Hover a dot" hint. Score uses text styling (not the circle treatment) to feel like a peek, not a result. On mobile/touch: tap activates the same behavior.
• View Results → — Text CTA drives click-through to full Module Results.
• Design rationale: The dot array creates a visual fingerprint. The hover detail rewards curiosity with progressive disclosure — one DP at a time, never the full picture. The score + name create a question ("why did I score 39 on Portfolio Analytics?") that only Module Results can answer.
Status Indicators — §5.1.2
• Not Started — gray score placeholder (—), muted text, no diagnostic data
• In Progress — amber progress bar (X of Y answered), amber left border, "Continue Assessment →" CTA, "Auto-saved" note
• Completed — score circle, band label, DP Dot Array (band-colored circles), "View Results →" CTA

Assessment Flow Rules — §5.3, §5.9
• One question per screen. No multi-question pages. No momentum breaks between questions.
• Three question types: Likert/BARS (primary — ranked options), Yes/No (binary cards), Slider (continuous scale). No open text in v1.
• Auto-save: On answer selection for Likert/Yes-No (auto-advances). Slider requires explicit "Next" tap after positioning.
• "Save & Exit": Always available. Returns to Module Home with In Progress state.
• Back navigation: Previous button preserved. Answers retained when navigating back.
• Resume behavior: Lands on next unanswered question. Progress count shown in progress bar.
• Nav sidebar: Accessible but visually dimmed (40% opacity) during assessment to reduce distraction.
• Completion screen (§5.3.3): Success confirmation + score preview ("Mirror" moment) + "View Your Results" CTA → Module Results with Three-Act Reveal.
Micro-Reward Animation Spec (v1.2 Enhancement — Dev Handoff)
On answer submit (Likert/Yes-No):
• T+0ms: Selected option border transitions to var(--accent), background fills to var(--accent-glow) (150ms ease-out)
• T+150ms: Green checkmark icon (✓) fades in next to progress count (200ms ease-in). Text updates: "7 of 12 → 8 of 12"
• T+350ms: "Saved" micro-label appears below progress bar, 10px, var(--green), fades out after 800ms
• T+500ms: Auto-advance to next question with 200ms slide-left transition
• Total interaction time: 700ms

On answer submit (Slider):
• Same checkmark + "Saved" sequence, but NO auto-advance (slider requires explicit "Next" tap)
• "Next →" button pulses once (scale 1.0→1.05→1.0, 300ms) after save confirmation

On module completion:
• T+0ms: Progress bar fills to 100% (400ms ease-out), color transitions from var(--accent) to var(--green)
• T+400ms: Large checkmark (✓) scales up from 0→1 with bounce easing (500ms). font-size:36px; color:var(--green)
• T+900ms: "Assessment Complete" text fades in (300ms)
• T+1200ms: Score circle scales up from 0→1 with spring easing (600ms). Band color revealed on completion.
• T+1800ms: "View Your Results" CTA fades in and pulses once
• Total celebration sequence: 2400ms

Three-Act Reveal animation timing (Module Results page):
• Act 1 (Score + Band): Immediate on page load. Score circle counts up from 0→final in 800ms
• Act 2 (IA sections): Stagger in at 300ms intervals per IA block. Fade-up from 20px below (400ms each)
• Act 3 (Action Plan + Moenio Lens): Fade in 500ms after last Act 2 element
• One-time animation only. Stored in session. Subsequent visits = instant final state.
DD-4: Question Type Visual Rendering
• Decision made in this wireframe: Likert uses stacked option cards (full-width, left-aligned text). Yes/No uses side-by-side cards with larger text. Slider uses horizontal track with thumb, value labels, and live readout.
• All types centered in main content area, max-width 600px for optimal readability
• Consistent progress bar, navigation controls, and module name across all types
• Question text is 16px/600 weight for clear hierarchy above answer options
