◇ Empty State Matrix §13 — Every page's lifecycle states
▼
Universal Pattern — All Pages
● Pre-assessment: Page structure shown with guidance text. CTA to begin first assessment. Shows what will appear.
● Partial completion: Available data displayed. Remaining sections: "Complete [X] to see [Y]." No blank sections.
● Full completion: All components active with full data.
● Tier-locked: Scores always visible. Intelligence depth varies. One upgrade prompt per section, not per item.
Page Pre-Assessment State Key Guidance
Dashboard Welcome expanded. Exec Summary shows ranking placeholder. Layer 3 shows Module Cards with start CTAs. Quick Win, Moenio Nudge, New Intelligence hidden until first module. Estimated time to first result (15-20 min). Encouragement to begin.
My Results Message per tab: "Complete a module to see [scores / strengths / trends]." Link to Assessments section.

◇ Screen Build Specs v2.0 — Acceptance criteria and data requirements per screen
▼
Global Build Rules
Layout: Two-panel (left nav + center content). Desktop-first. Left nav collapses to hamburger on mobile. No right panel.
Positioning: Diagnostic only. Never coaching, prescriptive, or advisory language. "Your Action Plan" is the only near-advisory element — uses prompts, not directives.
Tier Model: Hybrid. Same structure, different depth. Scores never locked. One upgrade CTA per section, not per item. No lock icons on individual scores.
Voice System: V1 (Gray/Teal) for platform analysis, V2 (Gold) for client composite, V3 (Green) for external benchmark. Visual distinction is mandatory.
Naming: "Your" prefix in UI, omit in structural/API. P93-UNS v3 identifiers for all elements. Area codes are permanent.
BI Differentiation: BI screens must be visually distinct from module results (§7.4). Different card style, color treatment, or layout pattern.
